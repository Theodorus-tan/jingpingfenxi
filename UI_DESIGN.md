# AI 竞品分析系统 — UI 设计规范

> 基于 Arco Design Pro 改造，对标字节系竞赛技术栈标准。

---

## 一、页面架构

### 1.1 路由与菜单

| 路由 | 页面 | 说明 | Icon |
|------|------|------|------|
| `/` | 起始页 | 进入竞品分析入口 | — |
| `/dashboard/workbench` | 工作台 | 概览统计 + 最近分析 | `IconDashboard` |
| `/competitors/list` | 竞品管理 | 竞品列表、搜索、CRUD | `IconList` |
| `/analysis/new` | 新建分析 | 输入竞品，启动 Agent | `IconSettings` |
| `/analysis-report/view` | 分析报告 | AI 报告展示 | `IconFile` |

### 1.2 菜单结构

```
AI 竞品分析系统
├── 📊 工作台        /dashboard/workbench
├── 🔍 竞品管理      /competitors/list
├── 🧠 新建分析      /analysis/new
└── 📋 分析报告      /analysis-report/view
```

## 二、页面设计

### 2.1 起始页 `/`

极简居中卡片布局，深色渐变背景。

```
┌──────────────────────────────────────┐
│                                      │
│            AI 竞品分析系统             │
│       深度洞察竞对动态，助力业务决策     │
│                                      │
│         [ 进入竞品分析 ]              │
│                                      │
│         AI 竞品分析系统               │
└──────────────────────────────────────┘
```

- 背景：`linear-gradient(163.85deg, #1d2129 0%, #00308f 100%)`
- 卡片：居中，`padding: 60px 80px`，圆角 `12px`
- 按钮：圆角 `24px`，`200x48px`
- 点击后跳转至 `workbench` 页面

### 2.2 工作台 `/dashboard/workbench`

单列卡片布局。

```
┌──────────────────────────────────────┐
│  [快捷操作]                            │
│  新建分析  竞品管理  查看报告           │
├──────────────────────────────────────┤
│  [概览]                               │
│  竞品数: 23  分析: 128  评分: 8.2  新增: 5│
├──────────────────────────────────────┤
│  [最近分析] (从 localStorage 获取)     │
│  DJI Osmo Action 4   运动相机  已完成  │
│  GoPro Hero 13       运动相机  已完成  │
│  ...                                 │
└──────────────────────────────────────┘
```

### 2.3 竞品管理 `/competitors/list`

```
┌──────────────────────────────────────┐
│  [搜索框]    [品类筛选]    [添加竞品]   │
├──────────────────────────────────────┤
│  竞品名称    品类      状态    操作    │
│  ──────────────────────────────────── │
│  DJI Action 4  运动相机  ✅ 已分析  分析│
│  GoPro 13      运动相机  🔄 分析中  等待│
│  ...                                  │
└──────────────────────────────────────┘
```

交互：
- 搜索框防抖 300ms
- 品类下拉筛选
- 弹窗表单添加竞品
- "分析"按钮跳转到新建分析页

### 2.4 新建分析 `/analysis/new`

```
┌──────────────────────────────────────┐
│  [竞品名称输入框]                      │
│  ┌──────────────────────────────┐    │
│  │ 例如：DJI Osmo Action 4      │    │
│  └──────────────────────────────┘    │
│  [开始分析]  [重置]                   │
├──────────────────────────────────────┤
│  [分析师工作台] (分析进行时/完成后显示) │
│                                      │
│  🤖 开始分析任务...                    │
│  🔍 搜索：GoPro Hero 13 参数          │
│  ✅ 找到 5 条结果                      │
│     ├ GoPro 官网                      │
│     ├ Wikipedia 条目                  │
│  🤖 综合分析中...                      │
│  ✍️ 报告生成完成                       │
│                                      │
│  [查看报告]                           │
└──────────────────────────────────────┘
```

关键交互：
1. 用户输入竞品名称，点击"开始分析"
2. Agent 实时日志通过 SSE 推送展示（替代假进度条）
3. 完成后 localStorage 保存报告，跳转报告页

### 2.5 分析报告 `/analysis-report/view`

```
┌──────────────────────────────────────┐
│  GoPro Hero 13    AI 生成报告         │
│  生成时间：2026-05-12 10:35           │
├──────────────────────────────────────┤
│  [AI 深度洞察]                        │
│                                      │
│  (Markdown 渲染区域)                  │
│                                      │
│  # 竞品分析报告：GoPro Hero 13        │
│                                      │
│  ## 执行摘要                          │
│  GoPro Hero 13 是 GoPro 最新旗舰...   │
│                                      │
│  ## 竞品简介                          │
│  ...                                 │
│                                      │
│  ## 核心功能                          │
│  - 5.3K @ 60fps 视频录制              │
│  - HyperSmooth 6.0 防抖              │
│                                      │
│  ## SWOT 分析                        │
│  | 优势 | 劣势 |                      │
│  |------|------|                     │
│  | ...  | ...  |                     │
└──────────────────────────────────────┘
```

- 使用 `marked` 库将 Markdown 渲染为 HTML
- 没有 ECharts 图表，纯文字报告
- 数据来源：`localStorage`（latest_analysis_report）

## 三、布局规范

### 3.1 页面布局

```
┌──────────────────────────────────────┐
│  Navbar (64px)                       │
├───────┬──────────────────────────────┤
│       │                              │
│ Menu  │  Content                     │
│       │  (padding: 20px)             │
│ 220px │                              │
│       │  ┌──────────────────────┐    │
│ 可折叠 │  │ 卡片 (border: 8px)   │    │
│       │  │ padding: 20px        │    │
│       │  └──────────────────────┘    │
├───────┴──────────────────────────────┤
│  Footer                              │
└──────────────────────────────────────┘
```

### 3.2 间距

| 元素 | 值 |
|------|------|
| 内容区 padding | `16px 20px` |
| 卡片间距 | `16px` |
| 卡片内边距 | `20px` |
| 卡片圆角 | `8px` |
| 按钮圆角 | `4px` |

## 四、主题配色

使用 Arco Design 默认 Token 系统：

| 使用场景 | Token / 值 |
|---------|------------|
| 主色调 | `--color-primary-6: #165DFF` |
| 页面背景 | `--color-fill-2: #F2F3F5` |
| 卡片背景 | `--color-bg-2: #FFFFFF` |
| 主文字 | `--color-text-1: #1D2129` |
| 次文字 | `--color-text-2: #4E5969` |
| 辅助文字 | `--color-text-3: #86909C` |
| 成功/已完成 | `rgb(var(--green-6))` |
| 进行中 | `rgb(var(--arcoblue-6))` |
| 搜索图标 | `rgb(var(--orange-6))` |

## 五、组件状态

### Agent 日志条目状态

| 类型 | Icon | 颜色 | 说明 |
|------|------|------|------|
| thinking | `IconSend` | arcoblue | Agent 正在思考/规划 |
| searching | `IconSearch` | orange | 正在搜索信息 |
| search_result | `IconCheckCircle` | green | 搜索到结果 |
| writing | `IconEdit` | purple | 正在生成报告 |
| 等待中 | `IconLoading` | — | 正在处理 |
